# vibe-coding-guide

写给**不懂代码、靠 AI Agent 开发**（Vibe Coding）的人的一个 Claude Skill。

> Vibe Coding：用自然语言指挥 AI Agent（Claude Code / Codex / Cursor 等）写代码，你专注“要做什么”，把“怎么写”交给 AI。

这个 skill 不替你写代码，而是在你用 AI 搭建 / 改造 / 重构项目时，**替你把关、验证、用大白话解释**——把住安全与数据红线，避免“一改就崩”“换电脑就报错”“明文存密码”“金额用浮点”等常见坑。

## 它能帮上忙的场景

- **项目起步**：想做个网站 / 小程序 / App / 桌面工具 / 后台，不知从何下手
- **出了问题**：代码一改就崩、换电脑报错 / 白屏、满屏红字看不懂、AI 说改好了其实没好
- **数据与安全**：数据误删找不回、字段对不上、密码 / 敏感信息怎么存、会不会被拖库
- **上线交付**：老项目大检查、能不能上线、部署与 HTTPS

## 目录结构

```
vibe-coding-guide/
├── SKILL.md                  # 技能主文件（名称 + 触发描述 + 铁律 / 红线 / 工作流）
└── references/
    ├── handbook.md           # 完整方法论：三阶段 + 可复制提示词 + 5 类项目适配
    └── checklist.md          # 交付前 19 条自检清单
```

采用**渐进披露**：日常只加载 SKILL.md（精简、常驻），需要细节时才按需读 references 里的文件，省上下文。

## 如何安装

把 `vibe-coding-guide/` 整个文件夹放到 Claude 的 skills 目录：

- 个人全局：`~/.claude/skills/vibe-coding-guide/`
- 或某个项目内：`<项目>/.claude/skills/vibe-coding-guide/`

放好后，Claude 在识别到上面的场景时会自动调用本技能。

## 设计原则

- **四条协作铁律**：删改数据 / 装依赖先问；同一问题修 3 次就停下分析；不准只说“我修好了”，要拿证据；重要决定写进项目真源文档（CLAUDE.md / AGENTS.md）。
- **八条安全红线**：前端不碰库、后端独立鉴权、密码强哈希、金额非浮点、敏感信息加密脱敏、关键数据软删除 + 审计、密钥不进 Git / 前端、默认口令强制改。

---

本技能以“让不懂代码的人也能安全地用 AI 开发”为初心。它不替你写代码，而是给你一套判断、验证和不被误导的方法。欢迎提 issue / PR。
